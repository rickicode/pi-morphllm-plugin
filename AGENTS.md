# Repository Instructions

- Do not create or update `CHANGELOG.md` immediately when a change is still tentative, exploratory, or not yet final.
- Before creating or updating `CHANGELOG.md`, always ask the user for confirmation first.
- When a major change is confirmed as final, create or update `CHANGELOG.md` so the version history records what changed.
- Do not require confirmation before ordinary repository edits unless the user specifically asks for it.
